    1. What is Semantic HTML?
	Semantic HTML is a standard used to markup webpages. 

    2. What is HTML used for? 
	HTML is used to give structure to a web page. 

    3. What is an attribute and where do we put it? 
	Attributes are specific names for elements, such as classes or IDs

    4. What is the h1 tag used for? How many times should I use it on a page?
	The h1 tag is a headline, probably used only once.

    5. Name two tags that have required attributes
	"input" and "select" are tags that have required attributs.

    6. What do we put in the head of our HTML document? 
	Metadata goes in the head. Metadata is data that the user isn't meant to see or care about. 

    7. What is an id?
	An id is an attribute with a high level of specificity. 
 
    8. What elements can I add an id to?
	Id attributes can be added to any element.
 
    9. How many times can I use the same id on a page?
	You can only use an id once. 
 
    10. What is a class?
	A class is an attribute with less specificity than an id.
 
    11. What elements can I add a class to? 
	Class attributes can be added to any element.

    12. How many times can I use the same class on a page?
	You can use a class only once, but generally many elements will have the same class.
 
    13. How do I get my link to open in a new tab?
	Add a target="_blank" attribute to your link (anchor tag).

    14. What is the alt attribute used for? 
	The alt attribute is used if the primary attribute cannot be used, for whatever reason.

    15. How do I reference an id?
	#"idname".

    16. What is the difference between a section and a div
	Section defines a group of elements that are related to a single theme, div is used for stylistic purposes only.

    17. What is CSS used for? 
	CSS is used to add style to a webpage.

    18. How to we select an element? Example - every h2 on the page
	h2{}. Type the element and add curly brackets. 

    19. What is the difference between a class and an id? - Give me an example of when I might use each one
	An id is like a class, but has more specificity. Use classes in general, and use id to target a very specific element, possibly an element that already has a class.

    20. How do we select classes in CSS?
	We select classes using dot notation, so .class-name{}

    21. How do we select a p element with a single class of “human””?
	p.human{}

    22. What is a parent child selector? When would this be useful?
	Child selectors are buried within the parent selector.
	Calling the parent selector will also call the child selector, but we can still call the child selector seperately
 
    23. How do you select all links within a div with the class of sidebar?
	.sidebar{}

    24. What is a pseudo selector?
	Pseudo selectors select elements based on different conditions, for example, the last element in a page.

    25. What do we use the change the spacing between lines?
	The line-height command in CSS can change spacing between lines.

    26. What do we use to change the spacing between letters?
	The letter-spacing command in CSS can change spacing between letters.

    27. What do we use to to change everything to CAPITALS? lowercase? Capitalize?
	In CSS we can type text-transform: uppercase, lowercase, and capitalize, respectively. 

    28. How do I add a 1px border around my div that is dotted and black?
	div{
	  border: 1px dotted black;
	}
    29. How do I select everything on the page? 
	*{} is a universal selector.

    30. How do I write a comment in CSS?
	You can use double slash characters to comment in CSS. // or /* */

    31. How do I find out what file I am in, when I am using the command line? 
	pwd tells you what file you are in.

    32. Using the command line - how do I see a list of files/folders in my current folder?
	Type ls to see a list of folders in your current folder.

    33. How do I remove a file via the command line? Why do I have to be careful with this?
	The command rm removes a file. There is no safeguard to warn you before deletion, so caution is needed.
 
    34. Why should I use version control? 
	Version control allows multiple branches of code to be written, enabling experimention and simultaneous work among multiple developers.
 
    35. How often should I commit to github?
	Commit to github as often as you would save your work, so every 20mins give or take. 

    36. What is the command we would use to push our repo up to github? 
	git push -u origin firstName-lastName.

    37. Walk me through Lambda's git flow.

	modify code, git add, git commit -m "Add comments", git push -u origin firstName-lastName, select personal branch in gitHub, submit pull request with TL as reviewer. Profit.
